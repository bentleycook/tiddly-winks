# tiddly-winks

Feature space manager for local dev. Named tmux sessions, dynamic ports, Caddy `*.localhost` routing, git worktrees, isolated databases, and bead-backed context that survives context loss.

## Concept

`tw start <feature>` creates a complete isolated environment:

- **Git worktrees** for each repo (branch `bentley/<feature>` in each)
- **Isolated database** via project-local `db_setup` hook
- **Services** running from the worktrees in tmux windows
- **Claude window** — dedicated orchestrator seat, pre-loaded with context
- **Bead** accumulating context, wrap-up history, PRs, and branch names across all sessions

The bead replaces manual `/wrap-up` → paste. Context loss is a non-event: wrap-up appends to the bead, new Claude reads the bead.

## Install

```bash
git clone git@github.com:bentleycook/tiddly-winks.git ~/Programming/tiddly-winks
cd ~/Programming/tiddly-winks
./install.sh
```

`install.sh` is idempotent — run it again after `git pull` to pick up any structural changes. The `tw` binary is symlinked, so code changes take effect immediately without reinstalling.

## Update

```bash
git -C ~/Programming/tiddly-winks pull
# Only needed if install.sh has changed (plist, new dirs):
~/Programming/tiddly-winks/install.sh
```

## Usage

```bash
tw start <feature>              # Create feature space
tw stop <feature>               # Stop services (keeps bead open)
tw stop <feature> --done        # Stop + tear down worktrees + close bead
tw list                         # Active sessions
tw status <feature>             # Ports, URLs, worktrees, bead
tw open <feature>               # Open primary URL in browser
tw claude <feature>             # Attach to Claude orchestrator window
tw append <feature>             # Append wrap-up to bead (reads stdin)
tw prune                        # Remove stale sessions after reboot
```

## Config file: `.tw.yml`

Place in workspace root. Also accepts `.devenv.yml` for compatibility.

```yaml
project: myproject

issue_tracker:
  type: linear       # or: github, none
  team: FAI          # Linear team key (if type: linear)
  # repo: owner/repo # GitHub repo (if type: github)

worktrees:
  base: worktrees              # worktrees created at worktrees/<feature>/<service>
  branch_prefix: bentley/      # branches named bentley/<feature>

services:
  frontend:
    path: my-frontend          # path relative to workspace root
    command: "bun run dev -- --port {port}"
    port_hint: 5173
    window: frontend

  backend:
    path: my-backend
    command: "pipenv run python runner.py"
    port_hint: 5001
    env: "PORT={port}"
    window: backend
    db_setup: "./dev/db.sh switch"   # run after worktree created

proxy:
  "": frontend        # http://myproject-<feature>.localhost → frontend
  api: backend        # http://myproject-<feature>-api.localhost → backend
```

Override service paths locally without committing:

```yaml
# .tw.override.yml (gitignored)
services:
  backend:
    path: ../my-backend-local
```

## Context-loss recovery

When Claude's context fills up:

1. Run `/wrap-up` in Claude — copy the output
2. `tw append <feature>` — paste + Ctrl-D
3. `tw claude <feature>` in a new terminal — paste the printed context block
4. New Claude reads the bead: `bd show <bead-id>`

## Architecture

```
~/.tiddly-winks/          runtime state (not in repo)
  sessions.json           active sessions, ports, worktree paths
  Caddyfile               auto-generated from sessions.json
  logs/caddy.log
  com.bentley.tw-caddy.plist   generated by install.sh

~/Programming/tiddly-winks/   this repo
  bin/tw                  the CLI (symlinked to ~/.local/bin/tw)
  install.sh              idempotent installer
  system/
    caddy.plist.template  template — install.sh fills in paths

workspace/
  .tw.yml                 project config
  worktrees/
    <feature>/
      <service>/          git worktree, branch bentley/<feature>
```

## Requirements

- macOS (uses Caddy + launchctl for `*.localhost` routing)
- [Caddy](https://caddyserver.com/docs/install) (`brew install caddy`)
- tmux
- Python 3 + PyYAML (`pip3 install pyyaml`)
- [beads](https://github.com/bentleycook/beads) (`bd` CLI)
- `~/.local/bin` in PATH
